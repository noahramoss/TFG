Plataforma web para la gestión de finanzas personales
Este Trabajo Fin de Grado se centra en el diseño e implementación de una plataforma web para registrar, categorizar y analizar ingresos y gastos. La solución adopta una arquitectura cliente–servidor con API REST (Django + Django REST Framework) y una SPA en React, lo que facilita su evolución y la integración con otros clientes o servicios.
La aplicación ofrece un panel de control con indicadores clave (ingresos, gastos y balance) y visualizaciones por periodo. El usuario puede definir categorías personalizadas (ingreso/gasto), aplicar filtros por fecha, tipo y categoría, así como buscar y ordenar movimientos. Las reglas de negocio (por ejemplo, importes positivos y unicidad de categorías por usuario y tipo) garantizan la coherencia de los datos en todo momento.
La API REST asegura un contrato claro entre frontend y backend e incluye paginación, búsqueda y ordenación en los listados principales. La autenticación basada en token protege los recursos y permite un flujo de navegación con rutas privadas en el frontend. Para el almacenamiento se utiliza SQL (ruta rápida con SQLite y opción de PostgreSQL para entornos más exigentes), manteniendo integridad referencial y rendimiento suficiente para el alcance del proyecto.
El diseño de la interfaz prioriza la simplicidad y la lectura rápida: KPIs superiores, filtros visibles, tablas consistentes y formularios breves (con fecha por defecto “hoy” en nuevos movimientos). Este enfoque busca minimizar fricción y favorecer el uso cotidiano desde ordenador.
Las instrucciones detalladas para poner el sistema en marcha (entorno, dependencias, variables, arranque de backend y frontend) se encuentran en el capítulo “Manual de instalación e implementación” de la memoria.
